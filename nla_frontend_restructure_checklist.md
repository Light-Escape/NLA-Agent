# 数值代数工作台前端重构清单

目标是把当前前端升级为一个**简洁、优雅、有趣**，并且真正适合数值代数任务的工作台。

## 1. 先把信息结构理顺

### 必做
- 按会话隔离 workspace，每个聊天记录对应独立的变量空间、历史状态和文件引用。
- 区分三层内容：
  - 聊天层：用户和 AI 的自然语言对话。
  - 状态层：矩阵、向量、分解结果、诊断量等真实对象。
  - 展示层：Markdown、LaTeX、解释性文本。
- 让 workspace 成为“状态真相”，中间聊天输出只是解释，不是最终状态来源。

### 目标效果
- 用户一眼看出这次计算属于哪个会话。
- AI 和用户看到的是同一份结构化状态，不会串台。

## 2. 重构右侧 Workspace：从“变量列表”升级为“数学对象面板”

### 必做
- 每个对象统一卡片化，展示名称、类型、尺寸、存储方式、来源、版本、更新时间。
- 给对象加语义标签，如 `matrix / vector / factorization / residual / diagnostic / preconditioner`。
- 增加状态标签，如 `fresh / derived / stale / invalid / pinned`。
- 对象组显示，例如 `SVD(A)` 自动聚合 `U, Σ, Vt`，`LU(A)` 自动聚合 `L, U, P`。

### 建议加
- 对象详情抽屉：点击变量后右侧展开详细信息，不打断主视图。
- 一键固定重要对象：例如把 A、当前残差、最终解固定在顶部。

### 目标效果
- workspace 看起来像一个“数学实验台”，而不是普通调试面板。

## 3. 增加“依赖关系”和“版本链”

### 必做
- 记录每个对象的来源：来自用户输入、文件导入、工具计算，还是 Agent 推导。
- 记录对象依赖：例如 `x` 来源于 `solve(A,b)`，而 `A` 来源于文件 `foo.mat`。
- 支持版本管理：同名变量可保留多个版本，不直接覆盖。
- 支持回滚：某次分解结果出错，可以退回上一版。

### 目标效果
- 后面做自动推理、自动纠错、自动重算时会非常顺。

## 4. 聊天区改成“解释 + 任务流”而不是纯文本

### 必做
- 支持 Markdown + LaTeX 的稳定渲染，优先保证公式、编号公式、矩阵环境正确。
- 把 AI 输出拆成结构化消息类型：
  - 解释
  - 计算
  - 结论
  - 警告
  - 需要用户确认
- 对长输出增加折叠，避免一大段推导淹没界面。

### 建议加
- 消息头显示“这条消息做了什么”，例如：读取文件、构造预条件子、计算残差、输出结果。
- 错误消息可点击展开，查看原始报错、上下文和建议修复动作。

### 目标效果
- 聊天区像“推理报告”，而不是单纯聊天记录。

## 5. 加一个“数值代数专用任务栏”

### 建议做成顶部或输入框附近的小区域
- 常用任务快捷入口：
  - 解线性方程组
  - 最小二乘
  - 特征值问题
  - SVD
  - 稀疏分解
  - 预条件
  - 迭代法
- 任务模板：每种任务自动给出推荐输入格式和常见诊断项。

### 目标效果
- 用户不用每次都从自然语言重新组织问题。
- AI 也更容易进入正确的算法路径。

## 6. 给数值对象增加“诊断面板”

### 必做
- 显示数值代数核心指标：残差范数、相对误差、条件数估计、正交性误差、秩估计、收敛步数。
- 诊断结果高亮：正常、警告、失败要有明显区分，但不要刺眼。
- 支持对比：例如“本次迭代 vs 上次迭代”的残差变化。

### 目标效果
- 工作台不只是展示结果，而是展示“结果是否可信”。

## 7. 加一点“有趣”，但保持克制

这里的“有趣”不是花哨，而是让数值计算过程有一点探索感。

### 建议加
- 对象 hover 的轻反馈：鼠标悬停时轻微浮起，显示简短摘要。
- 状态变化动效：新对象出现、版本更新、计算完成时用柔和过渡。
- 结果卡片的“完成感”：例如 solve 成功后，结果卡片像被“放入工作台”的一件成果。
- 问题卡片 / 任务卡片：把一次计算包装成一张卡，像实验记录一样清晰。

### 注意
- 动效只做“轻”，不要做成游戏 UI。
- 趣味性应该来自“计算对象被整理得很漂亮”，而不是颜色乱跳。

## 8. 视觉风格统一成“简洁、优雅、安静”

### 必做
- 统一卡片圆角、阴影、间距。
- 减少视觉噪音，少用重边框，多用留白和层级。
- 统一字体层级：标题、变量名、元信息、说明文字要分层清楚。
- 控制颜色数量：主色 + 状态色 + 中性色即可。

### 建议风格
- 主界面像“研究工作台”。
- 细节像“高质量数学软件”。
- 气质像“安静但专业”。

## 9. 让 AI 真正“看懂前端”

### 必做
- 把前端状态导出成 JSON schema，包括 chats、sessions、workspace objects、dependencies、file references、ui state。
- 记录操作事件流：用户发言、文件上传、对象创建、变量更新、报错、回滚。
- 给 AI 提供结构化上下文，不只给截图，还要给组件树、状态快照、接口定义。

### 目标效果
- AI 不只是“看图猜”，而是真正能理解前端结构并给出可执行优化建议。

## 10. 推荐的重构顺序

### 第一阶段：打基础
- 会话隔离
- workspace 独立
- 对象卡片化
- Markdown + LaTeX 稳定渲染

### 第二阶段：增强可用性
- 依赖关系
- 版本链
- 诊断面板
- 任务模板

### 第三阶段：增强体验
- 动效
- 卡片聚合
- 折叠与摘要
- 更优雅的空状态和加载态

### 第四阶段：面向 AI 自动优化
- JSON 状态导出
- 操作日志
- 结构化上下文输入
- 自动截图/快照联动

## 一句话定位

这个前端最终应该变成：

**一个会话隔离、对象可追踪、诊断可解释、界面安静优雅的数值代数实验工作台。**

最值得优先修的是：**会话隔离 + workspace 对象模型 + 依赖/版本管理**。
